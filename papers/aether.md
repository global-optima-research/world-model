# Aether: Geometric-Aware Unified World Modeling

> InternRobotics / OpenRobotLab, 2025
> ICCV 2025 & ICCV 2025 RIWM Outstanding Paper
> 论文: https://arxiv.org/abs/2503.18945
> 项目页: https://aether-world.github.io/
> 代码: https://github.com/InternRobotics/Aether
> 模型: https://huggingface.co/AetherWorldModel/AetherV1

---

## 一句话总结

Aether 是一个基于 CogVideoX-5B 的几何感知统一世界模型，通过 task-interleaved 多任务训练框架，在合成数据上联合优化 4D 动态重建、动作条件视频预测和目标条件视觉规划三大能力，实现了从合成数据到真实场景的零样本泛化，在深度估计和视频预测上达到甚至超越领域专用模型的性能，获得 ICCV 2025 RIWM Outstanding Paper。

---

## 核心贡献

1. **统一的几何感知世界模型框架**: 首次将 4D 动态重建（深度 + 相机位姿）、动作条件视频预测和目标条件视觉规划统一到单一视频扩散模型中，通过共享特征实现跨任务协同增益。

2. **Task-Interleaved 训练策略**: 提出任务交织特征学习方法——训练时动态遮掩不同任务的输入，使重建、预测、规划三个任务在特征空间中互相增强，而非简单的多任务混合。

3. **全合成数据训练 + 零样本真实泛化**: 构建全自动 4D 合成数据标注流水线，模型完全在合成数据上训练，却能在真实世界 Sintel、BONN、KITTI、ScanNet、TUM Dynamics 等数据集上零样本泛化，重建性能可比甚至超越领域专用模型。

4. **Raymap 相机表示创新**: 用 Raymap（射线图）表示相机轨迹，替代传统的离散相机参数，提供几何信息丰富的动作条件化方式，与扩散架构天然兼容。

5. **两阶段训练 + 多模态损失设计**: Stage 1 在 latent space 优化标准扩散损失；Stage 2 在 image space 引入 MS-SSIM（彩色视频）、SSI（深度视频）和 Pointmap（射线图 3D 对齐）三个专用损失。

## 方法详解

### 整体架构

```
输入: 观测图像 + [可选] 相机轨迹动作 / 目标图像
       ↓
[CogVideoX-5B 视频扩散模型 (微调)]
       ↓
   ┌───────────────┬──────────────────┬─────────────────┐
   │  4D 动态重建   │  动作条件视频预测  │  目标条件视觉规划  │
   │ (RGB-D + Pose) │ (未来帧生成)      │ (路径规划)        │
   └───────────────┴──────────────────┴─────────────────┘
```

### 三大任务

#### 1. 4D 动态重建
- 输入完整视频序列，估计每帧深度和相机位姿
- 输出: 深度图序列 + Raymap（隐式编码相机位姿）
- 可进一步转换为动态点云

#### 2. 动作条件视频预测
- 输入初始观测图像 + 相机轨迹动作（Raymap 形式）
- 输出: 符合给定相机运动的未来帧序列
- Raymap 提供几何一致的动作条件化

#### 3. 目标条件视觉规划
- 输入观测图像和目标图像对
- 输出: 从观测到目标的视频路径规划
- 无需显式动作指令，模型自主规划路径

### 关键创新

#### Raymap 表示
- 将相机内外参编码为逐像素的射线方向图
- 每个像素存储其对应的 3D 射线方向
- 与视频帧具有相同的空间分辨率，可直接拼接输入扩散模型
- 相比离散参数化（平移/旋转），提供更丰富的几何信息

#### Task-Interleaved 训练
- 训练时对不同任务的输入通道进行动态遮掩
- 重建任务: 提供完整视频，遮掩动作/目标
- 预测任务: 提供首帧 + 动作，遮掩其余帧
- 规划任务: 提供首帧 + 末帧，遮掩中间帧和动作
- 共享的 DiT 特征空间让三个任务互相增强

#### 合成数据标注流水线
- 基于 Kubric 等合成环境生成 RGB-D 视频
- 全自动标注流程:
  1. **Dynamic Masking**: 使用 Grounded-SAM-2 区分动态物体和静态场景
  2. **Camera Parameter Estimation**: 基于静态特征估计可靠的相机参数
  3. **Raymap Generation**: 从相机参数生成逐像素射线图

### 两阶段训练

#### Stage 1: Latent Space 优化
- 在 CogVideoX-5B 的 latent space 中进行标准扩散损失训练
- 学习视频生成和几何表示的基础能力

#### Stage 2: Image Space 精炼
- **MS-SSIM Loss**: 针对彩色视频，优化多尺度结构相似性
- **SSI Loss**: 针对深度视频，采用 Scale-and-Shift-Invariant 损失处理深度尺度模糊问题
- **Pointmap Loss**: 将预测的 disparity 和 raymap 转换为 3D 点云，最小化与 GT 点云的几何偏差

## 实验结果

### 视频深度估计

| 数据集 | 方法 | Abs Rel ↓ | δ < 1.25 ↑ |
|--------|------|-----------|------------|
| Sintel | ChronoDepth | - | - |
| Sintel | DepthCrafter | - | - |
| Sintel | DA-V | - | - |
| **Sintel** | **Aether** | **0.314** | **97.7** |
| Sintel (重建) | MonST3R-GA | 0.378 | - |
| **Sintel (重建)** | **Aether** | **0.324** | - |

- 在 Sintel 数据集上取得最优深度估计性能
- 显著优于 ChronoDepth、DepthCrafter、DA-V 等扩散式方法
- 重建性能优于专用几何方法 MonST3R-GA

### 消融实验
- **Aether vs Aether-no-depth**: 在两种评估协议下，完整模型一致优于去除深度任务的变体，验证了多任务协同的有效性
- Task-interleaved 训练 vs 简单多任务混合: 交织训练带来显著性能提升

### 相机位姿估计
- 在 Sintel、TUM Dynamics、ScanNet 数据集上评估
- 精度相对较低，可能是 raymap 表示与预训练视频扩散模型存在兼容性问题

### 零样本泛化
- 完全在合成数据上训练
- 在 Sintel、BONN、KITTI、ScanNet、TUM Dynamics 等真实数据集上零样本评估
- 重建性能可比甚至超越在真实数据上训练的领域专用模型

## 对 MambaWorld 的启发

1. **几何感知是世界模型的核心升级方向**: Aether 证明将 3D 几何理解（深度、相机位姿）融入世界模型可以大幅提升模型的物理真实性和泛化能力。MambaWorld 应考虑在架构中集成几何感知能力。

2. **Raymap 表示的借鉴**: 用逐像素射线图编码相机运动，比离散参数化更适合与生成模型结合。MambaWorld 可以探索类似的连续几何表示作为动作条件。

3. **多任务协同训练的价值**: Task-interleaved 训练让重建、预测、规划互相增益。MambaWorld 可以探索类似策略——例如让视频预测和深度估计共享 Mamba 特征空间。

4. **合成数据的巨大潜力**: 纯合成数据训练即可零样本泛化到真实场景，大幅降低数据收集成本。MambaWorld 可以利用类似的合成数据流水线快速迭代。

5. **CogVideoX-5B 作为基线参考**: Aether 基于 5B 参数的视频扩散模型微调，证明预训练视频模型是强大的世界模型起点。MambaWorld 可以探索用 Mamba 架构蒸馏或替换这类大模型中的注意力模块。

6. **两阶段训练策略**: Latent space 预训练 + Image space 精炼的策略值得借鉴——先学习粗粒度表示，再用专用损失优化细粒度质量。

7. **Pointmap Loss 的启发**: 将预测转化为 3D 点云再计算损失，是一种优雅的几何一致性约束。MambaWorld 如果引入几何任务，可以采用类似的损失设计。

## 局限性 / 研究机会

1. **高动态场景表现差**: Aether 在大幅运动或密集人群等高度动态场景中表现不佳。这是基于扩散模型的世界模型的共性问题，Mamba 的高效序列建模可能有助于更好地处理快速变化。

2. **相机位姿估计精度有限**: Raymap 表示与预训练视频扩散模型存在兼容性问题，导致位姿估计不够稳定。研究更好的几何条件化方式是机会。

3. **非实时推理**: 基于 5B 参数的扩散模型，推理速度远低于实时交互需求。与 Matrix-Game 2.0 的 25 FPS 形成鲜明对比。MambaWorld 有机会实现"几何感知 + 实时交互"的结合。

4. **合成到真实的域差距**: 虽然零样本泛化效果不错，但在复杂真实场景中仍有性能下降。混合训练策略（合成 + 少量真实数据）可能进一步提升。

5. **缺少显式物理建模**: 几何感知 ≠ 物理感知。模型不具备显式的物理引擎能力（碰撞检测、力学模拟）。将几何感知与物理模拟结合是重要研究方向。

6. **固定分辨率和帧率**: 当前模型在固定分辨率和帧数下工作，缺乏灵活的时空适应能力。探索可变分辨率/帧率的统一世界模型是机会。

7. **规划能力较初步**: 目标条件视觉规划只是生成从 A 到 B 的视频路径，缺少显式的决策推理或避障能力。将 world model 与 policy learning 深度结合是未来方向。
